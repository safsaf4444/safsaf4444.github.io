As a biomedical student I kept running into the same problem: research evidence is spread across multiple databases, hard to compare quickly, and even harder to appraise without a structured workflow. I wanted to build a tool that made that process faster, clearer, and more transparent — and to actually learn how to build software in the process.

Flagship project

SEREN

● Live Active development
The problem: medical research is distributed across multiple databases, inconsistently formatted, and often difficult to evaluate without statistical confidence or a structured review process. SEREN brings together search, appraisal, synthesis, and organisation in one place — without requiring institutional access or a paid subscription to get started.
Live — serenresearch.com GitHub
i.

Multi-Source Search

Searches across Europe PMC, OpenAlex, PubMed, Crossref, ClinicalTrials.gov, and preprint sources in a single workflow. Results are normalised and deduplicated using DOI matching, PMID fallback, and title-based comparison.

ii.

AI Evidence Appraisal

Supports structured PICO extraction, evidence-focused summaries, and appraisal workflows. Uses a multi-provider AI setup with free-tier fallback so the platform remains accessible without a paid API key.

iii.

Cross-Paper Synthesis

Select papers from your library and generate a structured synthesis — agreement, disagreement, evidence gaps, overall takeaways. Designed for comparison across multiple sources rather than isolated reading.

iv.

Statistical Calculators

NNT/NNH, Odds Ratio with 95% CI, Cohen's d, p-value interpreter, and confidence interval visualiser — entirely in the browser, no login required.

v.

Personal Library

Save papers to folders, add notes and highlights, track reading status, and export citations in 8 formats including BibTeX, APA 7th, Vancouver, and Harvard.

vi.

Testing & Deployment

93 automated tests covering auth, deduplication, provider parsing, and API routes. GitHub Actions CI on every push. Deployed on Vercel with Neon PostgreSQL.

What the SEREN codebase covers

Authentication systemJWT, PBKDF2, Google OAuth2, optional auth pattern

Data project

NeuroTrials Explorer

● Live
The project: an interactive dashboard exploring 27,000+ clinical trial records across 7 neurological conditions — stroke, Alzheimer's disease, Parkinson's, MS, ALS, migraine, and epilepsy. Built to strengthen my understanding of APIs, data cleaning, dashboarding, and neuro-related health data workflows.
Live dashboard GitHub
i.

ETL Pipeline

Fetched 27,000+ trials via a paginated REST API with retry logic and exponential backoff. Raw JSON saved per condition; separate cleaning step flattens nested data into a CSV.

ii.

Pandas Cleaning

Handled messy real-world data — inconsistent sponsor names, missing phases, unparseable dates — with vectorised pandas operations. Documented every limitation honestly.

iii.

6 Interactive Charts

Plotly Express line, grouped bar, stacked bar, donut, and horizontal bar charts — each a standalone function decoupled from the dashboard and testable independently.

iv.

Streamlit Dashboard

Deployed on Streamlit Community Cloud with sidebar filters for condition, year range, status, and phase. Reactive execution — every widget interaction re-runs the full filter chain.
